üü¢ Netlify Deployment

Method 1: Deploy Without Git (Drag & Drop)

1 Create a ZIP file:
  • Select all your portfolio files (index.html, style.css, script.js, images folder)
  • Create a ZIP archive (don't zip the parent folder, zip the contents)
  • Make sure index.html is in the root of the ZIP
2 Deploy to Netlify:
  • Go to netlify.com
  • Click "Deploy to Netlify for free"
  • Drag and drop your ZIP file to the deploy area
  • Wait for deployment (usually 1-2 minutes)
3 Get your live URL:
  • Netlify will generate a random URL like: https://amazing-name-123456.netlify.app
  • You can customize this URL in Site Settings ‚Üí Change site name

Method 2: Deploy With Git (Continuous Deployment)

1 Create GitHub Repository:
# Initialize git in your project folder
git init
git add .
git commit -m "Initial portfolio commit"

# Create repository on GitHub and push
git remote add origin https://github.com/yourusername/your-portfolio.git
git branch -M main
git push -u origin main
2 Connect to Netlify:
  • Sign up/Login to Netlify
  • Click "New site from Git"
  • Choose GitHub and authorize Netlify
  • Select your portfolio repository
3 Configure Build Settings:
Build command: (leave empty)
Publish directory: (leave empty or put "/")
Branch: main
4 Deploy!
  • Click "Deploy site"
  • Netlify will build and deploy your site
  • Future pushes to GitHub will auto-deploy

‚ö° Vercel Deployment

Method 1: Deploy Without Git

1 Install Vercel CLI:
# Install Vercel CLI globally
npm install -g vercel

# Or use without installing
npx vercel
2 Deploy from terminal:
# Navigate to your portfolio folder
cd your-portfolio-folder

# Deploy to Vercel
vercel

# Follow the prompts:
# - Set up and deploy? Yes
# - Which scope? (your account)
# - Link to existing project? No
# - What's your project's name? your-portfolio
# - In which directory is your code located? ./
3 Your site is live!
  • Vercel will provide a live URL
  • For production deployment, run: vercel --prod

Method 2: Deploy With Git

1 Push to GitHub:
# If you haven't already, create a GitHub repo
git init
git add .
git commit -m "Initial portfolio commit"
git remote add origin https://github.com/yourusername/portfolio.git
git push -u origin main
2 Connect to Vercel:
  • Go to vercel.com
  • Sign up with GitHub
  • Click "New Project"
  • Import your portfolio repository
3 Configure and Deploy:
Framework Preset: Other
Root Directory: ./
Build Command: (leave empty)
Output Directory: (leave empty)
Install Command: (leave empty)
  • Click "Deploy"
  • Vercel will build and deploy your site
  • Future commits will trigger automatic deployments

üåê Custom Domain Setup

For Netlify:

1 Add Custom Domain:
  • Go to your site dashboard on Netlify
  • Click "Domain management"
  • Click "Add custom domain"
  • Enter your domain (e.g., yourname.com)
2 Configure DNS:
# Add these DNS records at your domain provider:
Type: A Record
Name: @ (or root)
Value: 75.2.60.5

Type: CNAME
Name: www
Value: your-site-name.netlify.app

For Vercel:

1 Add Domain:
  • Go to your project dashboard on Vercel
  • Click "Domains"
  • Enter your custom domain
  • Follow the DNS configuration instructions

‚ö†Ô∏è Important: DNS changes can take 24-48 hours to propagate globally. SSL certificates are automatically generated once DNS is configured.

üîß Troubleshooting

Common Issues and Solutions:

üîç Site Not Loading

  • Make sure index.html is in the root directory
  • Check that all file paths are correct (case-sensitive)
  • Verify images folder is included in deployment

üñºÔ∏è Images Not Showing

  • Check image paths in HTML (should be relative: images/profile.jpg)
  • Ensure images folder is uploaded
  • Verify image file names match exactly (case-sensitive)

üì± Contact Form Not Working

  • Netlify: Forms work automatically, add netlify attribute to form tag
  • Vercel: Requires serverless function or external service (like Formspree)

üåê Custom Domain Issues

  • DNS changes take time (24-48 hours)
  • Use DNS checker tools to verify propagation
  • Ensure DNS records are configured correctly

üöÄ Performance Optimization

  • Compress images before uploading
  • Minify CSS and JavaScript files
  • Enable Brotli compression (automatic in both platforms)
